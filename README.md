## Что это? 

Это консольный проект, содержащий несколько скриптов для генерации памяток для сценариев в игре Кровь на часовой башне (https://boardgamegeek.com/boardgame/240980/blood-on-the-clocktower, https://bloodontheclocktower.com/). 
Так, в проекте есть скрипты для генерации   
- Листа с перечислением всех ролей и способностей в сценарии    
- Памяток для рассказчика с порядком первой ночи и последующих ночей      
- Листа с перечислением всех дополнительных сказочников, странников и летописцев и упрощённым порядком ночей  
Короче говоря, это аналог официального инструмента по созданию сценариев: https://script.bloodontheclocktower.com/    

## Почему не официальный инструмент?    

Функционал этого приложения отличается от функционала официального инструмента в следующих моментах:    
1) Поддержка форматирования: выделение текста цветом, жирным шрифтом или курсивом.  
2) Переведённые на русский minion info, demon info, dusk и dawn, причём minion info и demon info даже с красивыми картинками 
3) Поддержка кастомных сценариев и ролей - в целом это есть и в официальном приложении, но с нюансом: в таком случае в скрипт автоматически добавляется пресловутый бутлегер даже если вы просто решили перевести существующую роль на русский язык! 
4) Поддержка кастомных сглазов - в официальном приложении можно добавить или обновить один из сглазов только с помощью всё того же бутлегера, невозможно даже перевести уже существующие сглазы на русский язык без лишнего правила бутлегера!   
5) На листе для дополнительных ролей (сказочников, странников и летописцев) есть возможность напечатать упрощённый порядок ночи, что определённо будет очень удобно игрокам  
6) Ну и очевидный плюс: open source  

## Как запускать    

Для запуска вот потребуется любой компилятор Python. Лично я использую PyCharm. Не буду уделять внимание объяснение тому, что такое Python и как запускать программы, написанные на нём, этой информации полно в интернете  
1) Клонируете репозиторий   
2) Скачиваете командой в командной строке из корня проекта pip install -r requirements.in все зависимости (единожды для проекта) 
3) Добавляете в корень проекта файл с JSON вашего сценария с названием script.json (о нём позже) 
4) Запускаете по очереди скрипты нужные вам скрипты из папки generators (например, в корне проекта в командной строке python generators/generate_roles.py).  

Результат: новенькие файлы в корне проекта.   
Внутри каждого скрипта в самом начале вы найдёте несколько констант. Для каждой константы есть небольшой комментарий, описывающий, зачем эта константа нужна. Изменение этих констант будет полезно для изменения визуальной составляющией результирующего файла.
Какой-то адекватный фронтенд для проекта не планируется.    

## О самих скриптах 

Все скрипты находятся в папке generators    
Один запуск скрипта даёт на выходе один pdf-файл    
Скрипты следующие:  
- generate_roles: генерирует самое главное - лист со всеми ролями и описаниями их способностей; подразуемвается, что такой лист должен быть в наличии у каждого игрока (в т. ч. у рассказчика)  
- generate_firstnight и generate_othernight: генерируют листы порядков ночей для первой ночи и последующих ночей соответственно, эти листы в подробностях описывают рассказчику, что надо делать ночью и в каком порядке; подразумевается, что такие листы должны быть только у рассказчика, причём их можно расположить на разных сторонах одного листа бумаги  
- generate_additional: генерирует перечисление всех сказочников, летописцев и странников, упрощённые порядки ночей (в которых не написано описание взаимодействий ролей с рассказчиком ночью, только картинки) и красивое (наверное) название сценария. В нём так же есть флаг NO_SIDE_COLUMNS, при значении true которого этот скрипт не выводит на странице упрощённые порядки ночей; предполагается, что это обратная сторона листа с ролями     

## Прочие файлы и директории проекта    

- assets: несколько изображений, использующихся в проекте вне зависимости от сценария   
- cache_images: сюда будут кэшироваться изображения для ролей из открытых источников (подробнее позже)    
- fonts: шрифты. По умолчанию Dumbledor - шрифт для почти всего текста, adanascript - шрифт для названия сценария в скрипте generate_additional. Если появится желание использовать какой-то другой шрифт, надо будет изменить константы для шрифтов в скриптах (суммарно 5 штук). Если планируется писать текст сценария на русском языке, рекомендую подобрать шрифты, поддерживающие русский язык. 
- json_examples: примеры уже отформатированных и заполненных JSON-объектов для многих официальных ролей. При составлении собственных сценариев можно использовать и править эти схемы. Но важно отметить, что в каких-то места могут содержаться ошибки разного уровня значимости и что ещё не для всех ролей присутствуют примеры JSON. Работа над этим пока в процессе  
- .gitignore, README.md, requirements.in - стандартные штуки для любых репозиториев git (хотя подозреваю, что если вы дочитали до этого момента, вы сами знаете)  
- script.json - в исходном репозитории просто пример готового JSON-файла сценария, как раз его вы в основном и будете править при создании собственных сценариев (аналогично тому, как это работает в официальном инструменте)
- order.py - выводит в консоль отсортированный по определённому принципу список названий ролей из файла script.json, об этом речь пойдёт позже    
- unformat.py - создаёт на основе script.json новый json без форматированного текста и массивов jinxes, нужен для того, чтобы получить из файла json, пригодного для этого проекта, файл json, пригодный для онлайн-гримуара

## JSON сценария    

JSON сценария - документ определённого формата, в котором содержится вся информация о сценарии. Это массив словарей. Каждый словарь относится либо к метаинформации о сценарии (если значение его поля id равно _meta), либо к определённой роли, присутствующей в сценарии (горожане, изгои, приспешники, демоны, сказочники, странники и летописцы)   
Первым элементом массива лучше сделать следующую вещь:  
{   
    "id": "_meta",  
    "name": "name",   
    "author": "author"  
}   
name - название сценария, author - его автор. Название сценария и его автор в качестве заголовков используются в генераторах. Для примера выше roles, firstnight и othernight будут использовать строку name by author, additional будет использовать строку name (без имени автора). Эти поля можно оставить пустыми. Для названия сценария в каждом скрипте есть собственная константа, которая используется только в том случае, если в элементе с id _meta это поле отсутствует     
Далее перечислены роли, они выглядят примерно так:

Подробнее о всех полях:
- id: в этом приложении никак не используется, но полезно в некоторых других проектах по BotC (например, здесь: https://github.com/bra1n/townsquare/tree/main)  
- name: название роли 
- team: тип роли (townsfolk, outsider, minion, demon, fabled, traveler, loric - горожанин, изгой, приспешник, демон, сказочник, странник и летописец соответственно). ВАЖНО: не стоит допускать каких-либо ошибок или неточностей в этом поле, их строго всего 7 значений, и если как-то ошибиться в этом поле, роль банально не появится в результате.   
- firstNight и otherNight: отвечают за порядок сортировки ролей в первую и последующие ночи соответственно; это дробное число. Сортировка определена по возрастанию этих значений. Где-то в коде м Используются в firstnight, othernight и additional.     
- firstNightReminder и otherNightReminder: описания подсказок в первую и последующие ночи соответственно; используются в firstnight и othernight. 
- reminders: в этом приложении не используются, но очень полезны в уже упомянутом выше приложении для ведения онлайн-гримуара. Это жетоны напоминаний, которые будут присутствовать в функционале приложения для ведения онлайн-гримуара (со значком роли, в которой этот ремайндер прописан). Например, жетон "Ложного следа" Гадалки. 
- setup: в коде приложения никак не используется; это флаг, подсказывающий, влияет ли роль на начальное распределение ролей   
- ability: текст способности роли; участвует в roles. 
- image: ссылка на картинку роли из открытого источника. ВАЖНО: в данный момент поддерживаются только ссылки, выложите используемые вами картинки в открытые источники, чтобы далее использовать их в своём проекте   
- jinxes: перечисление всех сглазов. Представляет из себя словарь, ключи которого - строки с названиями ролей, с которыми у данной роли существует сглаз, а значение похоже на объекты, которые мы видим, например, в поле ability. Сглазы перечисляются в файле, получаемом из generate_additional. Так же миниатюры иконок ролей, лежащих в массиве jinxes, перечислены рядом с названием роли в файле из generate_roles. В общем, пример можно посмотреть, если добавить в сценарий, допустим, lleach и slayer. jinxes не терпит ошибок в названиях ролей.    

Обратите внимание, что лучше иметь две версии JSON-файла сценария: одну подходящую для этого проекта, другую подходящую для онлайн-версии гримуара. Отличаются они полями ability, firstNightReminder, otherNightReminder и jinxes. Для получения из отформатированного специально для этого проекта файла JSON сценария обычный файл JSON есть в корне проекта скрипт unformat.py. Он берёт JSON script.json из корня проекта и создаёт на его основе новый, упрощённый JSON тоже в корне проекта.    

## Форматирование текста    

В отличие от официального инструмента поля firstNightReminder, otherNightReminder и ability в JSON ролей - это массив объектов, а не одна строка.   
Каждый объект должен содержать поле text и может содержать поля color (значениями которого являются строки с названиями цветов, например, "blue", "red", "magenta", "green", "gold"), bold или italic (с булевыми значениями true/false), причём объект может содержать сразу несколько дополнительных полей.    
Итоговый текст, отображаемый для этой роли в итоговых PDF - это друг за другом записанный текст из каждого объекта с соответствующим форматированием.   
Таким образом, чтобы выделить в тексте какой-то кусок, понадобится разделить этот текст на три части:   
1) Всё, что идёт до подстроки, которую необходимо отформатировать;   
2) Сама подстрока с необходимым форматированием;  
3) Всё, что идёт после.  

Аналогично для всех кусков текста, которые необходимо отформатировать.    
В файлах в папке json_examples можно найти примеры того, как можно форматировать текст в способностях ролей.

## Изображения  

Все изображения в файлах директории json_examples взяты с русскоязычной вики: https://botc.1nv.is/  
Они все имеют одинаковый размер 591х591.    
Если вы собираетесь добавлять собственные изображения, позаботьтесь о том, чтобы размеры их основных частей - то есть иконок без пустой рамки - размером походили друг на друга, иначе одни изображения в итоговом варианте будут сильно меньше других. Возможно, когда-нибудь это исправлю  
Изображения не подтягиваются из локальных файлов. Чтобы использовать изображение, необходимо добавить в JSON ссылку на него в открытом источнике. Так что если собираетесь использовать кастомные изображения, позаботьтесь о том, чтобы выложить их в открытый источник.    
При первом обращении изображения кэшируются в папку cache_images. Крайне не рекомендуется переименовывать файлы в этой папке или добавлять что-то лишнее, это может очень быстро привести к мусору в директории.    
Изображения в папке assets - это изображения для иконок рассвета, заката, информации демонов и приспешников и иконки джинна (полезно для добавления сглазов). Как-то менять их имеет смысл исключительно если есть желание поменять эти изображения в финальном варианте.   
Для масштабов изображений в скриптах есть константы с названиями по типу IMAGE_SCALE. С помощью этих констант можно манипулировать размерами изображений для того, чтобы уместить все изображения на одной странице (вы же не хотите печатать по несколько страниц на каждого игрока).  
Так же позаботьтесь о том, чтобы у вашего изображения был прозрачный фон.   

## Порядок  

Порядок перечисления ролей зависит от того, в каком порядке они перечислены в файле JSON. То есть все роли внутри одной группы отсортированы в том порядке, в котором они перечислены в файле. Так что чтобы поменять порядок ролей на листе, поменяйте их порядок в JSON сценария. 

## SAO  

SAO (Standard Amy Order) - полезный принцип, по которому сортируются роли в сценарии. Я опишу этот принцип, по сути переведя информацию с этой страницы

1) В листе ролей сначала перечисляются все Горожане, затем Изоги, после Приспешники и, наконец, Демоны  
2) Внутри каждой группы роли отсортированы по типу способностей, так, официальный порядок по типам способности выглядит так (в порядке уменьшения приоритета):

Если у вас есть оригинал роли на английском и вы просто хотите её перевести, то можете ориентироваться при сортировке по способности оригиналов и сверяться с левым столбцов таблицы выше. Иначе сверяйтесь с чуть менее точным правым столбцом.    
Если у вас затесались две роли с одинаковым типом механик ролей, то раньше в списке появляется роль с более короткой способностью. Если и тут мимо, то раньше появляется роль с более коротким названием, а если даже тут ничья, то сортируются в алфавитном порядке.   

## Почему SAO важен?    

Формат SAO необязателен к использованию, но с ним вашим игрокам будет значительно проще ориентироваться в сценарии. 
Приложение не поддерживает никакой автоматической сортировки в соответствии с SAO. И никакой поддержки не планируется. Этим предлагается озаботиться вам. Но тем не менее небольшая поддержка в приложении предусмотрена, а именно скрипт order.py предназначен для вывода всех ролей из файла script.json в корне проекта по возрастанию длины описания способности, а роли с одинаковой длиной описания способности сортируются по возрастанию длины названия, а далее в алфавитном порядке. Таким образом вам не понадобиться самостоятельно вести подсчёт символов описаний способностей, если вы захотите использовать SAO.    

## Советы по созданию сценариев 

Советы по созданию сценариев от одного из разработчиков игры: https://www.reddit.com/r/BloodOnTheClocktower/comments/1cr2141/guidelines_for_script_building/    
Несколько устаревший, но всё ещё полезный список правил, взятый из проекта, созданного для проверки качества сценариев (стоит обратить внимание на поля error): https://github.com/botcscriptcheck/botcscriptcheck.github.io/blob/main/rules.json   

Советы в источниках выше очень простые, но одно из них стоит прокомментировать в контексте данного проекта, а именно число ролей.   
За предельно редкими исключениями сценарий должен содержать следующее число ролей каждого типа: 
1) РОВНО 13 Горожан;    
2) 4-5 Изгоев;   
3) 4-5 Приспешников; 
4) Не более 4 Демонов;   
5) Суммарно не более 25 ролей.   

Таким образом, если вы собираетесь добавить 5 Изгоев и 4 Приспешников (или наоборот) в свой сценарий, то более 3 Демонов добавлять в сценарий не рекомендуется. Если же вы хотите добавить 5 Изгоев и 5 Приспешников в сценарий, то не рекомендуется добавлять более 2 Демонов в сценарий.   
Это косвенно и отражается и на функционале приложения: если вы добавите чересчур много ролей, они банально не уместятся на одном листе. Вам вряд ли захочется печатать по два листа на каждого игрока, а вашим игрокам вряд ли захочется разбираться сразу в двух листах вместо одного. Так что вы, возможно, попробуете уменьшить размер изображений и размер шрифта. Но тогда информация на листе станет нечитабельной.    
Как быть? Всё очень просто: не добавлять больше 25 ролей в сценарий. Это не особенность реализации приложения и не моя прихоть, это совет разработчика игры.
